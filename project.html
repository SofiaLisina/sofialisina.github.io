Development of a mobile UI-kit for the pet walking service “Pawfect”

Pet-walking services are becoming increasingly in demand in large cities, where pet owners often lack the time for daily walks. Developing a mobile UI-kit for the "Pawfect" brand is highly relevant, as it will simplify user interaction with the service, making it more accessible and convenient. A well-designed UI-kit not only speeds up the development process but also enhances the quality of the final product, ensuring a positive user experience and increasing trust in the brand.

Goals

Creation a mobile user interface kit for the "Pawfect" brand, targeted at users seeking a convenient and reliable dog-walking service. This UI-kit should focus on user interaction, providing easy navigation and an appealing visual experience.

Solved tasks

  1. Analyzed existing solutions in the field of mobile applications for walking pets and identified their shortcomings.
  2. Determined the key elements that should be included in the UI-kit to ensure ease of use.
  3. Developed the concept and design of UI-kit elements in accordance with iOS guidelines, including icons, buttons, footer, walker profile cards and other components.

Implementation

The project is implemented as a set of components for Figma, which can already be used to create prototypes of mobile interfaces.

Project development stages

References

Websites and mobile apps which were used for an inspiration and selection of the color palette.
Typography and colors

Blue color palette is associated with confidence and calmness. Complimentary orange colors are used to attract an attention of the user.
Open Sans offers balanced shapes for excellent readability at any size, making it versatile for all interfaces.
Grid

The design follows iOS 18 guidelines, with horizontal dimensions optimized for a 4-column grid for proper scaling across screens. Element heights align with the modular grid—multiples of 8 pixels for large elements and 4 pixels for smaller ones—ensuring consistency and visual harmony.
Mi-Fi prototype

In the process of developing the UI-kit, Lo-Fi mockups were sketched on paper, and then screen mockups were laid out in Mi-Fi quality.
Icons

Icons were designed on a 32×32 px grid and placed within 40 px blocks. A 24×24 px set was also created for smaller UI elements.
Buttons

Main buttons are 48 px high, secondary buttons 32 px, both with an 8 px corner radius. Distinct contrasting colors highlight different states.
Inputs

Input field states are easily recognizable, with a minimal and restrained use of colors.
Navigation

A well-designed footer improves UX by simplifying navigation and making the interface more intuitive.
Various control elements and frames

Cards and controls are a key component of applications, used to display information in compact, block-style formats.
